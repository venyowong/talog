# 选用 vlang 开发的弊端

## 明显的缺点

- 没有 IDE
- 难以调试

## 何时使用引用，不够清晰

许多场景 vlang 都会自动使用引用，查看编译出来的 C 代码，可以看到变量都是指针类型。但是有些时候，vlang 会复制值，比如 `Closures`，可翻阅 [官方文档](https://github.com/vlang/v/blob/master/doc/docs.md) `Closures` 部分，若要在闭包内外同步数据修改，需要传递引用。但对于刚接触 vlang 的新手来说，文档可能没有看得那么仔细，一开始尝试 `Closures` 会遇到很诡异的问题：闭包内的数据修改无法体现在闭包外

有时候，vlang 会自动将 Struct 转为引用，这样 v 代码看起来用的是 Struct，导致开发者会误以为可以直接传递 Struct，然后在某些场景下，会出现"逻辑"上同一个变量，有不同的值，这个问题的本质就是底层代码读写的是不同的引用。要解决这种问题，需要开发者去注意到每个参数传递的细节，尽可能显式指定引用传递。
